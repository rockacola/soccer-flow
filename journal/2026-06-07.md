# 2026-06-07

## What I worked on

Redesigned the match timing model from elapsed-second counters to absolute timestamps.

## Decisions made

- Replaced `startedAt`, `elapsedSeconds`, and `segmentActualSeconds` with a single `segments: MatchSegment[]` array where each entry has `segmentType: 'period' | 'break'` and `createdAt: number` (absolute ms timestamp).
- All segment timestamps are pre-computed at match start and adjusted directly when the user corrects the schedule — no more relative arithmetic.
- Removed pause/resume entirely: time adjustments serve that need.
- Activities now store `createdAt: number` (absolute timestamp) instead of `elapsedSeconds`.
- `periodCount` is no longer stored — derived as `segments.filter(s => s.segmentType === 'period').length`.
- Screen re-rendering uses a local `useReducer` increment every second; the store holds no tick state.

## What I learned

Wall-clock absolute timestamps are the correct primitive for a match timer. Anything counter-based breaks under background/foreground app state transitions on iOS.

## Problems / friction

None — 98/98 tests green on first run after the rewrite.
