# 2026-06-05

## What I worked on

Set up the project from scratch: Expo 56 + TypeScript, NativeWind for styling, Zustand for state management, ESLint 9 + Prettier, and a layered architecture (UI → Services → Stores) designed to accommodate future backend integration without rewrites.

## Decisions made

- Expo managed workflow to avoid native build complexity
- Zustand over Context API — no providers, slice-level subscriptions
- NativeWind (Tailwind v3) for all styles — no StyleSheet
- AsyncStorage as local source of truth; cloud sync is additive when needed
- Services layer enforced from day one — screens stay dumb
- Repository layer deferred until a second storage implementation appears

## What I learned

- NativeWind v4 requires Tailwind v3 — v4 engine is not yet supported
- Expo 56 targets React Native 0.85; `react-native-screens@4.25+` requires RN >= 0.82
- `moduleResolution: "bundler"` is the correct tsconfig setting for Metro (replaces deprecated `node`)

## Problems / friction

- Several peer dependency version conflicts resolved during initial `npm install`
