# 2026-06-05

## What I worked on

Set up the project from scratch: Expo 56 + TypeScript, NativeWind for styling, Zustand for state management, ESLint 9 + Prettier, and a layered architecture (UI → Services → Stores) designed to accommodate future backend integration without rewrites.

## Decisions made

- Expo managed workflow to avoid native build complexity
- Zustand over Context API — no providers, slice-level subscriptions
- NativeWind (Tailwind v3) for all styles — no StyleSheet
- AsyncStorage as local source of truth; cloud sync is additive when needed
- Services layer enforced from day one — screens stay dumb
- Repository layer deferred until a second storage implementation appears

## What I learned

- NativeWind v4 requires Tailwind v3 — v4 engine is not yet supported
- Expo 56 targets React Native 0.85; `react-native-screens@4.25+` requires RN >= 0.82
- `moduleResolution: "bundler"` is the correct tsconfig setting for Metro (replaces deprecated `node`)

## Problems / friction

- Several peer dependency version conflicts resolved during initial `npm install`

## Next step

- Wire up React Navigation v7 bottom tabs
- Connect Zustand stores to AsyncStorage (load on mount, persist on change)

## Session 3

## What I worked on

Completed Phase 2: storage key constants, both stores wired to AsyncStorage via Zustand `persist`, `RootParamList` types, and React Navigation bottom tabs with two nested stack navigators.

## Decisions made

- `currentMatch` excluded from AsyncStorage persistence via `partialize` — transient state should not survive an app restart
- `loadTeams` and `loadPastMatches` removed; Zustand `persist` handles rehydration automatically
- Placeholder screens created for all five routes so navigation is runnable before Phase 3 UI work begins
- Added `@react-navigation/native-stack` (was missing from dependencies)

## What I learned

- `NavigatorScreenParams<T>` is required on the root tab param list to correctly type nested stack navigators
- `GestureHandlerRootView` must wrap the app and needs `flex: 1` to avoid collapsing to zero height

## Problems / friction

None.
