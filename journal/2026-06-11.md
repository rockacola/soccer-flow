## What I worked on

Added Montserrat as the app-wide custom font (100 Thin, 400 Regular, 500 Medium, 600 SemiBold, 700 Bold). Applied across all 24 StyleSheet files.

## Decisions made

Used `@expo-google-fonts/montserrat` (bundles .ttf files in the JS bundle, no native linking needed). Each weight is a separately named font family (`Montserrat_600SemiBold` etc.), so `fontWeight` is replaced with `fontFamily` everywhere — setting both is redundant and can misbehave on Android. The live-match timer keeps its monospace `fontFamily` (Menlo/monospace) for numeric readability. Font loading returns `null` during the brief load window; the splash screen covers this in production builds. Added `fonts` constant to `src/constants/typography.ts` alongside `typeScale`.

Test on simulator to confirm font renders as expected.

## Decisions made

Header-right text label ("Add") chosen over icon — font-based icons (Ionicons, expo-symbols) had glyph centering issues, and text labels are the standard iOS HIG pattern anyway. The iOS 17 UIBarButtonItem circular press highlight is accepted as native platform behaviour. Splash screen is a flat static image only — no React Native text overlay. `SplashScreen.preventAutoHideAsync` / `hideAsync` used in `App.tsx` to keep the native splash up while fonts load.

## Problems / friction

Font-based icons (Ionicons, expo-symbols) both had centering issues in the navigation header — glyph metrics not aligned within the em square. iOS 17 applies an automatic circular `UIBarButtonItem` background to custom header views; not suppressible from JS. Splash image requires a dev build (`expo run:ios`) — Expo Go ignores `app.config.js` splash config entirely. Gradient banding in `splash.png` is a PNG 8-bit export issue; fix is to add subtle noise/grain in Figma before export.

## What I worked on

Replaced floating action buttons (FABs) with header-right buttons on TeamsListScreen and TeamDetailScreen. Added a contextual empty-state "Add Team" button on TeamsListScreen.

## Decisions made

Used `navigation.setOptions({ headerRight })` via `useEffect` in each screen — the React Navigation idiomatic pattern for dynamic header options. Memoised `openAddModal` in `useTeamDetailScreen` with `useCallback` to keep it a stable dep. Tried `Ionicons` and `expo-symbols` for the icon but both had centering issues (font glyph metrics); settled on "Add" text label which is the standard iOS HIG pattern anyway. iOS 17 applies an automatic circular `UIBarButtonItem` background to custom header views — accepted as native platform behaviour, no code workaround.

## What I learned

Font-based icon libraries (`@expo/vector-icons`) have the same glyph metric centering issues as regular text. `expo-symbols` (SF Symbols) is the correct iOS-native solution but didn't resolve the issue here. The circular highlight on header buttons is UIKit's automatic button styling in iOS 17, not controllable from JS without code ugliness.
